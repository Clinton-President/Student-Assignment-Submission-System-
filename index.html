Mobile Assignment Submission System - UX Design Prototype

👥 User-Centered Design Research

Primary Personas

Alex - The Busy Student

Age: 20 | Year: Sophomore

Juggles part-time work and full course load. Needs quick, efficient submission process. Often submits assignments while commuting.

Pain Points: Time constraints, mobile-first needs, deadline anxiety

Maria - The International Student

Age: 22 | Year: Graduate

English as second language, prefers clear instructions and visual feedback. Uses various file formats from different software.

Pain Points: Language barriers, file compatibility, unclear requirements

Jordan - The Tech-Anxious Learner

Age: 35 | Year: Returning Student

Less comfortable with technology, needs intuitive interface. Values clear confirmation of successful submissions.

Pain Points: Tech anxiety, need for reassurance, prefer simple workflows

Key User Stories

  • As a busy student, I want to quickly see all my pending assignments so I can prioritize my work effectively.
  • As an international student, I want clear visual indicators of submission requirements so I don't make mistakes due to language barriers.
  • As a returning student, I want immediate confirmation when I submit an assignment so I know it was successful.
  • As any student, I want to track my submission history so I can reference past work and deadlines.
  • As a mobile user, I want to submit assignments from my phone so I can meet deadlines even when away from my computer.

🎯 HCI Principles Applied

1. Visibility & Feedback

Implementation:

  • • Clear due date indicators with color coding (red = urgent, yellow = upcoming)
  • • Progress indicators for file upload
  • • Immediate visual feedback for successful submissions
  • • Status badges showing submission state

This addresses Jordan's need for reassurance and reduces anxiety for all users.

2. Consistency

Implementation:

  • • Uniform color scheme throughout the app (blue gradient theme)
  • • Consistent card-based layout for courses and assignments
  • • Standardized navigation patterns and button styles
  • • Same iconography and typography across screens

Reduces cognitive load and learning time, especially beneficial for tech-anxious users like Jordan.

3. Accessibility & Inclusion

Implementation:

  • • High contrast color combinations for better readability
  • • Clear, simple language avoiding technical jargon
  • • Visual icons paired with text for universal understanding
  • • Large touch targets (minimum 44px) for easy mobile interaction

Particularly important for Maria (ESL) and ensures usability across diverse user groups.

4. Error Prevention & Recovery

Implementation:

  • • File format validation before upload
  • • Clear requirement displays to prevent submission errors
  • • Confirmation dialogs for important actions
  • • Helpful tips and contextual guidance

5. Mobile-First Design

Implementation:

  • • Touch-optimized interface with appropriate spacing
  • • Thumb-friendly navigation at bottom of screen
  • • Minimal cognitive load with focused, single-task screens
  • • Quick access to most common actions

6. Recognition Over Recall

Implementation:

  • • Dashboard shows all pending work at a glance
  • • Assignment requirements displayed during submission
  • • Visual progress indicators and status labels
  • • Contextual information always visible

🧪 User Testing Plan - If More Time Available

Key Areas to Test:

  • 📱 Mobile Usability Testing: Test with actual mobile devices to ensure touch targets are appropriate, text is readable, and navigation flows smoothly across different screen sizes.
  • 🕐 Task Completion Time: Measure how long it takes users to complete core tasks (finding assignments, uploading files, submitting) compared to existing systems.
  • 🎯 Error Rate Analysis: Track common mistakes users make during submission process, particularly around file format requirements and deadline interpretation.
  • ♿ Accessibility Testing: Test with users who have visual impairments, motor difficulties, or are non-native English speakers to ensure inclusive design.
  • 🔄 Cross-Platform Consistency: Verify the experience works equally well on iOS and Android devices, and test with users who switch between mobile and desktop.
  • 😰 Stress Testing: Observe user behavior when submitting assignments close to deadlines or when dealing with technical issues (poor network, file upload failures).
  • 🎓 Persona-Specific Testing: Conduct separate sessions with each persona type (busy students, international students, returning students) to validate design decisions.
  • 📊 Information Architecture: Test whether users can easily find specific assignments, understand the categorization system, and navigate between courses efficiently.

Testing Methods:

Moderated Mobile Testing Sessions: 30-minute sessions with 5-8 users per persona, using screen recording and think-aloud protocol.

A/B Testing: Compare current design against alternative layouts for key screens (dashboard organization, submission flow).

Longitudinal Study: Follow users over 2-3 weeks of actual assignment submissions to identify pain points that emerge with real usage patterns.
